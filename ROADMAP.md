# OmniaTools Roadmap

## Vision
Build a focused suite of lightweight media tooling apps under one repository, starting with caption and thumbnail workflows.

## Early-Stage Direction
1. **Establish a stable monorepo foundation**
   - Keep a clear separation between apps, shared packages, and documentation.
   - Define contribution and repository conventions early.
2. **Prototype product-focused applications**
   - Start with `omnia-caption` and `omnia-thumbnail` as independent apps.
   - Prioritize clear user flows before implementation complexity.
3. **Identify reusable core modules**
   - Extract shared utilities only when duplication appears.
   - Keep packages minimal and practical in the initial phase.
4. **Build delivery discipline**
   - Document tasks, milestones, and decisions in-repo.
   - Add CI/testing conventions once the first app scaffolds are ready.

## Short-Term Milestones
- M1: Repository structure and planning docs complete.
- M2: App-level technical scaffolds selected and documented.
- M3: First vertical slice implemented in each app.
- M4: Shared package strategy validated from real usage.
